array, object state 변경하는 법

숙제 해설 ~ 

숙제를 개발자 형식으로 바꿔서 작성
- array의 0번째 요소 = '여자 코트 추천';
- titleChange(글제목)
- but, 원본은 건드리지 마셈
- 복사본을 만들어서 진행
let copy = 글제목;
copy[0] = '여자 코트 추천';
titleChange(copy);
- but, 복사할 때 이렇게 하면 안되고 아래 처럼 해야해
let copy = [...글제목];
- 동작 원리
-1. state변경함수 특징: 괄호 안의 state로 기존의 state를 갈아치운다
-1.1. 기존 state == 신규 state -> 변경 안해줌
-2. array/object 데이터의 동작 방식 약간 특이
-2.1. ley arr = [1,2,3];을 하면 [1,2,3]을 갖는 array를 arr에 담아주는 것이 아니라, 그 array를 가리키는 곳(레퍼런스)을 arr에 담는다. [1,2,3]은 미지의 공간에 저장
-3. 같은 레퍼런스를 갖는 변수끼리는 등호로 비교해도 같다고 나옴
- 그래서 let copy = 글제목;으로 했을 때 titleChange(copy)를 해줘도 copy와 글제목은 같은 레퍼런스를 갖기 때문에 기존과 신규의 state가 같다고 판단해서 동작하지 않는 것이다.
**let copy = [...글제목]에서 ...의 이유:
spread operator라는 문법 -> array나 object 자료형 왼쪽에 붙일 수 있고 그 뜻은 괄호를 벗기라는 뜻이다. 간단히 말해 괄호 벗기기용 연산자이다.
array나 object 자료형을 복사할 때 많이 사용한다
let data1 = [1, 2, 3]
let data2 = [...data1];
console.log(data1 == data2); // false로 나온다
위 처럼 찍으면 data1의 괄호를 벗긴 1,2,3을 새로운 괄호로 덮기 때문에 data2의 레퍼런스가 data1의 레퍼런스와 다르다. 새로운 array로 인식하는 것이다.
그렇게 때문에 완전 독립적인 array 복사본을 생성해줄 수 있다.
여기서 독립적인 사본을 shallow copy 혹은 deep copy라고 한다.

오늘의 정리:
리액트에서 array/object의 state를 수정하고 싶다면
[...기존state] 와 같이 독립적인 카피본을 만들어서 수정하는 것이 좋다
