html 코드는 return(); 안에서 짠다
그리고 return(); 안에는 병렬로 태그 2개 이상 기입 금지

변수(let, var, const)처럼 자료를 보관할 때 state 사용 가능
- 상단에 import { useState } from 'react';
- useState(보관할 자료)
- let[작명, 작명]
ex. let[a, b] = useState('나는 정경호');
a: state에 보관했던 자료가 나옴
b: state 변경을 도와주는 함수(담 시간에 써봄)

state를 만들고 사용하는 것은 destructuring 문법
let num = [1, 2];
let a = num[0];
let c = num[1];
위 처럼 array의 요소를 담는 것도 가능하지만
let [a, c] = [1, 2];
위 처럼 등호의 왼쪽과 오른쪽을 짝짓는 방법으로도 가능하다

변수가 있는데 state를 왜  쓰는가?
- 일반 변수가 변경이 필요해서 JS에서 변수 내용을 바꿨어도 html에서 반영 안된다
- 변수 내용 바꼈으니 html도 고쳐주세요 라는 코드를 짜야함(쌩자바스크립트는 원래 그래야함)
- state는 위의 과정 없이 JS에서 수정을 하면 html에서 자동 재랜더링이 된다

다음 시간에 state랑 변수랑 변경하는걸 해보면 차이가 바로바로 눈에 보일거다

state를 쓰면 좋은 것과 아닌 것을 고려해보자
예를 들어 블로그의 제목과 로고가 있다고 치자
블로그의 제목과 로고가 자주 바뀌나? 아니다
state는 javascript 코드의 변경 사항이 html에 자동으로 재랜더링되는 기능을 사용하기 위해 쓴다
그렇기 때문에 자주 바뀌지 않는 제목과 로고 같은 것들은 그냥 변수에 담거나 하드코딩을 해도 된다

빡대가리식 정리:
자주 변경될 것 같은 html부분(제목, 내용, 클래스명)은 state로 만들어놓기
